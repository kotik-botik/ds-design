# Session Insights

Скилл из двух частей: (1) зафиксировать технические выводы сессии в MD, (2) предложить
оптимизации workflow через агентов/скиллы и спросить пользователя.

## Часть 1. Собрать и записать инсайты

1. **Восстанови, что реально делалось в сессии** — не по памяти, а по фактам:
   - `git log --oneline origin/main..HEAD` и/или `git log --since=...` — какие коммиты/PR.
   - `git diff --stat` ключевых правок; какие компоненты/файлы трогали.
   - Вспомни баги, которые чинили, и ПОЧЕМУ (корневую причину), а не только «что».

2. **СВЕРЬСЯ С ТЕКУЩИМ КОДОМ перед записью.** В этом репо часто идут параллельные
   правки/переименования (напр. `lenta-light.html` → `lenta-q3.html`, вынос логики в
   общий компонент). Прочитай актуальные файлы/`index.css`-импорты — не фиксируй
   устаревшие имена классов, хуков и путей.

3. **Запиши в `SESSION-INSIGHTS.md`** (в корне; обновляй существующий, не плоди дубли).
   Только полезное для следующего раза — «как делать правильно» + грабли. Группируй:
   - **Компоненты DS** — как использовать (разметка, модификаторы, нюансы).
   - **Кастомные компоненты** — когда DS не покрывает; принцип «одинаковая сущность ≠
     один компонент, если лейаут разный».
   - **Технические паттерны** — CSS/JS приёмы (ellipsis только на блоке → `__text`;
     `flex: 0 1 auto` чтобы мета липла к тексту; press-timing; и т.п.).
   - **Грабли** — что не работало и почему (напр. тайминг `pagereveal`; рывок от
     самописного transform на bfcache; VT-types не доезжают кросс-документно).
   - **Токены/иконки** — важные значения, slot-иконки vs `<img>`, currentColor.
   - **Техничка окружения/процесса** — allowlist, вложения vs инлайн-картинки,
     браузерная проверка анимаций агентом, и пр.
   - Каждую запись — кратко и конкретно (имя файла/класса/значение токена).

4. Закоммить и запушить по правилам проекта (ветка, PR, мерж) — как остальные изменения.

## Часть 2. Предложить оптимизации агентами/скиллами

5. Проанализируй сессию на **повторяющиеся многошаговые рутины** и подумай, что бы
   ускорило/обезопасило работу. Ориентиры (примеры, не догма):
   - Часто повторяемые цепочки (напр. commit → sync `-X ours` → PR → squash-merge →
     напоминание про деплой) → **скилл** «ship» одной командой.
   - Регулярная задача с внешним сервисом (Figma: get_design_context → download →
     allowlist-затык → фолбэк) → **скилл** «figma-asset» с готовым рецептом и фолбэками.
   - Проверка интерактива/анимаций в браузере → **агент** (уже есть `screenshot-testing`;
     предложить шаблоны-промпты под частые сценарии: переходы, dismiss, лайки).
   - Сверка вёрстки с Figma-нодой → **агент/скилл** «figma-diff».
   - Не предлагай агент/скилл там, где хватает одной команды.

6. Для каждого предложения дай: **что автоматизирует**, **что именно делает**,
   **триггер вызова**, и оценку выгоды (как часто всплывало в этой сессии).

7. **Задай пользователю вопрос** через `AskUserQuestion` (multiSelect): какие из
   предложенных агентов/скиллов завести сейчас. Первой опцией — самая высокоокупаемая
   (с пометкой «(Recommended)»). Если пользователь выберет — создай их
   (`.claude/skills/<name>/SKILL.md` или `.claude/agents/<name>.md`) и закоммить.

## Заметки
- MD пиши на языке проекта (здесь — русский), кратко и по делу.
- Не выдумывай «инсайты» ради объёма — только то, что реально всплыло и пригодится.
- Если `SESSION-INSIGHTS.md` уже есть — обновляй разделы, а не дублируй.
